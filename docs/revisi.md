### üî¥ MASALAH 2: Hero Section Terlalu Padat

**Masalah:**
- Hero section menggabungkan: Biggest Goal + Event Details + CTA + Speaker Preview
- Framework 1: Detail fokus hanya pada Biggest Goal di awal
- Speaker preview seharusnya muncul di section Features atau Proof/Authority (tapi Framework 1 tidak punya Proof/Authority)

**Dampak:**
- User langsung dihadapkan terlalu banyak informasi
- Fokus terpecah antara goal, detail acara, dan speaker
- Tidak sesuai dengan prinsip "fokus" di landing page

**Rekomendasi:**
1. **Hero Section - Simplifikasi:**
   - Fokus hanya pada Biggest Goal (title + deskripsi hook)
   - Pindahkan Event Details ke section terpisah atau ke bagian bawah
   - Pindahkan Speaker Preview ke section Features (sebagai bagian dari "siapa yang mengajar")
   - CTA di hero boleh tetap, tapi jangan terlalu menonjol (biarkan user baca dulu)

2. **Struktur Hero Baru:**
   ```
   - Title: "Rahasia Memilih Jurnal Scopus yang Tepat untuk Publikasi Anda"
   - Deskripsi hook (2 paragraf seperti sekarang) ‚úÖ
   - CTA sekunder (bukan utama): "Pelajari Lebih Lanjut" atau "Lihat Detail"
   - HAPUS: Event Details, Speaker Preview, CTA utama
   ```

**OPSI 2 - Pindahkan ke Posisi yang Tepat:**
- Jika tetap ingin mempertahankan, pindahkan ke SETELAH Features
- Atau munculkan sebagai bagian dari section Solution

**REKOMENDASI: OPSI 1** - Karena Framework 1: Detail fokus pada produk/solusi, bukan pada authority personal

### üî¥ MASALAH 4: Problem dan Solution Dicampur dalam Satu Section

**Masalah:**
- Framework 1: Detail memisahkan Problem dan Solution menjadi 2 section terpisah
- Landing page saat ini menggabungkan Problem dan Solution dalam 1 section dengan layout 2 kolom

**Dampak:**
- User tidak cukup "merasakan" masalah sebelum melihat solusi
- Alur persuasi tidak optimal (masalah harus "menggigit" dulu)
- Tidak sesuai dengan prinsip Framework 1: Detail

**Rekomendasi:**
1. **Pisahkan menjadi 2 Section:**

   **Section Problem (Baru):**
   - Title: "Masalah yang Sering Dihadapi Dosen & Peneliti"
   - Konten: 4 poin masalah (seperti sekarang)
   - Gaya: Narasi panjang, bukan hanya bullet point
   - Tambahkan: Kalimat empati seperti "Saya paham rasanya..."
   - Visual: Background merah muda/abu-abu, icon X merah

   **Section Solution (Baru):**
   - Title: "Workshop Ini Dirancang Khusus untuk Mengatasi Masalah Anda"
   - Konten: Perkenalkan workshop sebagai solusi
   - Gaya: Personal, praktis, kaitkan langsung ke masalah
   - Visual: Background hijau muda/biru muda, icon checkmark hijau

2. **Hapus layout 2 kolom side-by-side:**
   - Buat full-width untuk setiap section
   - Biarkan user fokus pada satu hal dalam satu waktu

### üî¥ MASALAH 5: Features Muncul Terlalu Awal

**Masalah:**
- Framework 1: Detail menempatkan Features SETELAH Testimoni
- Landing page saat ini menempatkan Features BERSAMAAN dengan Solution (di section "Apa Yang Akan Anda Pelajari?")

**Dampak:**
- Informasi teknis muncul sebelum trust terbangun
- User belum cukup yakin secara emosional sudah melihat features
- Tidak sesuai dengan prinsip "emotional first, rational later"

**Rekomendasi:**
1. **Pindahkan Section Features:**
   - Dari: Setelah Solution Section
   - Ke: Setelah Testimonials Section

2. **Restrukturisasi Section Features:**
   - Pisahkan dari section "Apa Yang Akan Anda Pelajari?"
   - Buat section terpisah dengan title: "Apa Saja yang Akan Anda Pelajari di Workshop Ini?"
   - Format: Bullet list dengan gaya narasi (bukan hanya poin-poin teknis)
   - Tambahkan: Penjelasan bahwa ini adalah "bekal praktis yang bisa langsung diterapkan"

### üî¥ MASALAH 6: Tidak Ada Statement + CTA Setelah Benefits

**Masalah:**
- Framework 1: Detail memiliki section "Statement + CTA" SETELAH Benefits
- Landing page saat ini langsung ke Special Offer setelah Benefits

**Dampak:**

   - Konten:

     ```
     karir akademik Anda yang akan membuka peluang publikasi di jurnal 
     Scopus berkualitas."
     
     [CTA: Daftar Sekarang - HANYA Rp. 99.000]
     ```
   - Gaya: Kalimat hook yang ringan tapi tegas
   - Visual: Background gradient atau card khusus dengan CTA button menonjol

### üî¥ MASALAH 7: Tidak Ada FAQ Section

**Masalah:**
- Framework 1: Detail memiliki section "Ask & Answer (FAQ)" SETELAH Special Offer
- Landing page saat ini TIDAK memiliki FAQ section

**Dampak:**
- Pertanyaan umum user tidak terjawab
- Kehilangan kesempatan untuk "pamer" keunggulan produk secara halus
- User mungkin ragu-ragu karena pertanyaan tidak terjawab

**Rekomendasi:**
1. **Tambahkan Section FAQ:**
   - Posisi: Setelah Special Offer, sebelum Statement Penutup
   - Format: Q & A dengan gaya ringan (bukan terlalu formal)
   - Contoh pertanyaan:
     - "Cocok untuk dosen/peneliti level apa?"
     - "Apakah materinya praktis dan bisa langsung diterapkan?"
     - "Bagaimana jika saya belum pernah publikasi di Scopus?"
     - "Apakah ada rekaman jika saya tidak bisa hadir?"
     - "Apakah ada garansi jika tidak puas?"
   - Tips: Campur FAQ informatif dengan "FAQ pancingan promosi"

### üü° MASALAH 9: Konten Problem Section Kurang Detail

**Masalah:**
- Framework 1: Detail menekankan Problem harus "sedetail mungkin" dengan gaya narasi
- Landing page saat ini hanya menampilkan 4 bullet point masalah

**Dampak:**
- User tidak cukup "merasakan" masalah
- Tidak ada empati yang kuat
- Tidak sesuai dengan prinsip "biar pembaca ngerasa: Saya banget!"

**Rekomendasi:**
1. **Perbaiki Konten Problem Section:**
   - Ubah dari bullet point menjadi narasi panjang
   - Tambahkan kalimat empati: "Saya paham rasanya..."
   - Struktur: Sebut masalah real ‚Üí Jelaskan penyebabnya ‚Üí Tunjukkan akibatnya
   - Contoh:
     ```
     "Artikel Anda sudah bagus, tapi selalu ditolak karena 'tidak sesuai 
     scope' atau 'tidak sesuai target audience'?"
     
     Saya paham rasanya.
     
     Anda sudah susah payah menulis artikel berkualitas, menghabiskan 
     waktu berbulan-bulan, tapi selalu ditolak. Bukan karena kualitas 
     artikelnya buruk, tapi karena salah memilih jurnal target. Akhirnya 
     Anda bingung mulai dari mana mencari jurnal yang tepat. Takut salah 
     pilih dan terjebak jurnal predatory. Sudah coba berbagai cara, tapi 
     tetap gagal menemukan jurnal yang cocok."
     ```

### üü° MASALAH 10: Konten Solution Section Kurang Personal

**Masalah:**
- Framework 1: Detail menekankan Solution harus "personal dan praktis"
- Landing page saat ini Solution terlalu singkat dan kurang mengaitkan ke masalah

**Dampak:**
- User tidak cukup "terhubung" dengan solusi
- Tidak jelas bagaimana solusi mengatasi masalah spesifik
- Tidak sesuai dengan prinsip "kaitkan langsung ke masalah"

**Rekomendasi:**
1. **Perbaiki Konten Solution Section:**
   - Perkenalkan workshop dengan format: "[Nama workshop] adalah [jenis] yang bantu [siapa] untuk [hasil]"
   - Kaitkan langsung ke masalah di section sebelumnya
   - Buat deskripsinya terasa personal dan praktis
   - Contoh:
     ```
     "Workshop 'Rahasia Memilih Jurnal Scopus yang Tepat' adalah panduan 
     praktis yang bantu dosen dan peneliti memilih jurnal Scopus yang 
     tepat untuk artikel mereka, sehingga artikel berkualitas tidak lagi 
     ditolak hanya karena salah memilih jurnal target."
     
     "Belajar lewat workshop 2 jam intensif, dengan tools dan teknik yang 
     langsung bisa dipraktikkan. Tidak perlu trial-error berbulan-bulan 
     lagi."
     ```

## CATATAN PENTING

1. **Framework 1: Detail cocok untuk produk digital/produk mahal** - Workshop ini termasuk kategori ini, jadi framework tepat digunakan.

2. **Prinsip utama Framework 1: Detail:**
   - Emotional first, rational later (Testimoni sebelum Features)
   - Problem harus "menggigit" dulu sebelum Solution
   - Multiple CTA untuk dorongan bertahap
   - FAQ untuk menghilangkan keraguan

3. **Elemen yang TIDAK ada di Framework 1 tapi perlu dipertahankan:**
   - Registration Form (wajib untuk konversi)
   - Footer (standar web)
   - Event Details (penting untuk workshop, bisa di form atau section terpisah)

4. **Elemen yang ada di Landing Page tapi TIDAK ada di Framework 1:**
   - Section "Belajar dari Para Ahli" (Proof/Authority) - Ini dari Framework 2/3
   - Rekomendasi: Hapus atau integrasikan ke Features

## KESIMPULAN

Landing page saat ini sudah memiliki konten yang baik, tapi **struktur dan urutan section tidak sesuai dengan Framework 1: Detail**. Perbaikan utama fokus pada:

1. **Reorganisasi urutan section** sesuai framework
2. **Pemisahan Problem dan Solution** menjadi section terpisah
3. **Penambahan section yang hilang** (Statement+CTA, FAQ, Statement Penutup)
4. **Perbaikan konten** menjadi lebih naratif dan personal
5. **Simplifikasi Hero Section** untuk fokus pada Biggest Goal

Dengan perbaikan ini, landing page akan lebih sesuai dengan Framework 1: Detail dan diharapkan dapat meningkatkan konversi sesuai target 10%+ visitor yang take action.
